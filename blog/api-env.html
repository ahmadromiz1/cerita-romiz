🔐 Mengamankan Akses API & Kunci Rahasia Menggunakan .env

Banyak pengembang pemula secara tidak sengaja membagikan kunci API atau informasi rahasia mereka karena tidak tahu cara menggunakan file .env. Artikel ini akan membahas praktik aman menyimpan informasi sensitif dalam proyek berbasis Git.

🚨 Masalah Umum: API Key Tercommit ke Publik

Banyak kasus di mana developer menyimpan API key langsung di dalam kode seperti ini:

API_KEY = "12345-abcde-fghij"

Ketika file ini dikirim ke GitHub, siapa pun bisa melihat kunci tersebut dan menyalahgunakannya. Bahkan bot crawler GitHub bisa mendeteksinya dalam hitungan menit.

🧪 Solusi: Gunakan File .env

  1. Buat file bernama .env di root proyek kamu.
  2. Masukkan variable rahasia ke dalamnya, contoh:
API_KEY=12345-abcde-fghij
DB_PASSWORD=my-secret-pass
  1. Gunakan package seperti python-dotenv (Python) atau dotenv (Node.js) untuk membacanya dalam kode.
  2. Tambahkan .env ke file .gitignore:
# .gitignore
.env

Dengan ini, Git tidak akan pernah mengikutsertakan file .env ke dalam commit.

📦 Contoh Implementasi (Python)

from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.getenv("API_KEY")

print(f"API kamu: {api_key}")

🔒 Tips Keamanan Tambahan

  • Jangan pernah commit .env, walau di private repo.
  • Gunakan layanan pengelolaan rahasia (secrets manager) untuk produksi.
  • Jika sudah terlanjur bocor, revoke API key sesegera mungkin.

📌 Bonus: Cara Menambahkan Secrets di GitHub Actions

GitHub menyediakan Secrets yang bisa digunakan di workflows. Masuk ke repo → Settings → Secrets → New repository secret.

Contoh penggunaan di action.yml:

env:
  API_KEY: ${{ secrets.API_KEY }}

🐍 Bonus untuk Pengguna Flask: Di Mana File .env Harus Ditaruh?

Dalam proyek Flask, file .env idealnya diletakkan di root proyek, sejajar dengan file seperti app.py atau run.py.

my-flask-app/
├── app.py
├── .env
├── requirements.txt
└── templates/

Contoh isi file .env:

FLASK_ENV=development
SECRET_KEY=sangat-rahasia
DATABASE_URL=sqlite:///site.db

Gunakan python-dotenv untuk membacanya di app.py:

from dotenv import load_dotenv
import os

load_dotenv()
secret = os.getenv("SECRET_KEY")

app = Flask(__name__)
app.config['SECRET_KEY'] = secret

📌 Pastikan juga menambahkan .env ke .gitignore agar tidak terupload ke GitHub.
